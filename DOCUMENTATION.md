# 📚 Полная документация JoinAuction

**JoinAuction** — это современное, высокопроизводительное решение для создания аукционов на серверах Minecraft версии 1.21.X. Плагин разработан с использованием Java 21 и асинхронных алгоритмов, что гарантирует отсутствие задержек (TPS drop) даже при огромном количестве лотов.

---

## 🛠 1. Конфигурационные файлы

### 📄 config.yml (Основное ядро)
Этот файл отвечает за техническую связь с базой данных, лимиты групп и визуальное оформление главного меню.

*   **settings.language**: Указывает, какой файл из папки `lang/` будет использоваться (например, `ru_ru`).
*   **settings.database**:
    *   `type`: `SQLITE` (локальный файл `auction.db`) или `MYSQL` (для синхронизации между серверами BungeeCord/Velocity).
    *   Настройки MySQL включают `host`, `port`, `database`, `username` и `password`.
*   **groups**: Секция для создания пресетов прав.
    *   `slots`: Лимит обычных предметов на продажу (`/ah sell`).
    *   `dsell-slots`: Лимит предметов для личного аукциона (`/ah dsell`).
    *   `time-hours`: Сколько часов предмет будет висеть на аукционе до истечения срока.
*   **menus.main-menu**: Настройка GUI.
    *   `size`: Размер инвентаря (строго кратно 9, максимум 54).
    *   `items`: Настройка кнопок управления. Каждая кнопка имеет `material`, `slot`, `name` и `lore`.
    *   *Важно:* Слоты 0-44 зарезервированы под отображение товаров.

---

### 📄 lang/ru_ru.yml (Локализация и Имена)
Файл управляет всеми текстовыми сообщениями и системой "Умных имен".

*   **prefix**: Текст, который вставляется перед каждым системным сообщением.
*   **item-settings**: Словарь имен.
    *   Если игрок выставляет предмет `STICK`, а в конфиге прописано `STICK: "Магический жезл"`, то во всех сообщениях о продаже будет фигурировать "Магический жезл".
    *   Поддержка формата `~[plugin:Name]` позволяет администратору оставлять пометки в конфиге, которые не видны игрокам.
*   **messages**:
    *   `sell-item-message`: Сообщение продавцу при выставлении.
    *   `item-sold-online`: Мгновенное уведомление продавцу, если он в сети.
    *   `item-sold-offline-summary`: Сводка прибыли, отправляемая игроку при входе на сервер.
*   **gui**: Настройка описания (Lore) предметов внутри аукциона.
    *   `item-lore-seller`: Вид предмета для того, кто его продает.
    *   `item-lore-buyer`: Вид предмета для всех остальных игроков.

---

### 📄 categories.yml (Категории товаров)
Настройка меню выбора категорий (`/ah categories`).

*   Каждая категория (например, `blocks`, `weapons`) имеет свой слот и иконку.
*   Плагин автоматически распределяет предметы по категориям на основе их материала.
*   Если предмет не подходит ни под одну категорию, он попадает в "Прочее" или отображается только во вкладке "Все товары".

---

### 📄 items.yml (Система поиска)
Расширяет возможности команды `/ah search`.

*   **search-tags**: Позволяет задавать ключевые слова (синонимы) для предметов.
    *   Пример: `DIAMOND: ["алмаз", "алм", "руда"]`.
    *   Если игрок введет `/ah search руда`, аукцион покажет все выставленные алмазы.

## 🔑 Система прав (Permissions)

Плагин поддерживает гибкую настройку через любой плагин прав (например, LuckPerms).


| Право | Описание |
| :--- | :--- |
| `auction.admin` | Доступ к команде `/ja reload` (перезагрузка всех конфигов). |
| `auction.serch` | Разрешает использовать поиск предметов `/ah search`. |
| `auction.player.serch` | Разрешает просматривать личные магазины через `/ah [ник]`. |
| `auction.group.[имя]` | Применяет лимиты из секции `groups` в `config.yml`. |
| `auction.slot.sell.[N]` | Устанавливает лимит слотов для `/ah sell`, где [N] — число. |
| `auction.slot.dsell.[N]` | Устанавливает лимит слотов для `/ah dsell`, где [N] — число. |
| `auction.slot.time.[N]` | Устанавливает время хранения лота в часах. |

## 🚀 Уникальные механики и особенности

### 📦 Полная поддержка NBT (ItemsAdder / Oraxen)
Благодаря использованию Base64-сериализации, JoinAuction сохраняет абсолютно все данные предмета:
*   Кастомные текстуры и модели (CustomModelData).
*   Сложные описания, созданные другими плагинами.
*   Специфические атрибуты (урон, скорость, здоровье).

### 💵 Система оффлайн-выплат
Если у игрока купили товар, пока он был оффлайн:
1. Деньги сохраняются в базе данных.
2. При следующем входе игрока плагин проверяет задолженность.
3. Выплачивает сумму через **Vault** и выводит красивое сообщение с количеством проданных предметов и итоговой прибылью.

### ⚡ Zero-Lag GUI (Асинхронность)
В отличие от старых плагинов, JoinAuction не делает запросы к базе данных в основном потоке сервера.
*   Открытие меню, перелистывание страниц и поиск происходят в фоновом режиме.
*   Это исключает "фризы" сервера даже при базе данных в 10 000+ предметов.

### 🛡️ Механика "Sold Placeholder"
Когда кто-то покупает товар, на его месте в GUI на несколько секунд появляется настраиваемый предмет (по умолчанию — серый краситель). Это предотвращает визуальные баги и дает игрокам понять, что слот только что освободился.

## 📝 Список доступных команд

*   `/ah` — Открыть главное меню.
*   `/ah sell [цена]` — Выставить предмет из основной руки.
*   `/ah dsell [цена]` — Выставить предмет на личный (скрытый) аукцион.
*   `/ah [ник]` — Открыть аукцион конкретного игрока.
*   `/ah search [запрос]` — Поиск по названию или тегам.
*   `/ah help` — Интерактивная справка по командам.
*   `/ja reload` — Перезагрузка (только для администраторов).

## 🧩 Интеграция с PlaceholderAPI

JoinAuction предоставляет собственные плейсхолдеры, которые можно использовать в Scoreboard, Tab, Holograms или других меню (требуется установленный PlaceholderAPI).


| Плейсхолдер | Описание |
| :--- | :--- |
| `%joinauction_active_slots%` | Текущее количество выставленных товаров игрока. |
| `%joinauction_max_slots%` | Общий лимит слотов (из прав или групп). |
| `%joinauction_expired_count%` | Количество товаров, ожидающих в хранилище. |
| `%joinauction_last_sale_price%` | Цена последнего проданного игроком товара. |
| `%joinauction_total_earned%` | Общая сумма, заработанная игроком на аукционе. |

## 🎨 Система визуализации (Цвета и Форматы)

### 🌈 HEX-Градиенты
Плагин поддерживает современный формат HEX-цветов. Вы можете использовать их в любом сообщении или названии предмета в `ru_ru.yml`.
*   **Формат:** `&{#RRGGBB}`
*   **Пример:** `&{#b50000}Текст` — создаст темно-красный цвет.
*   **Градиент (логика):** Чтобы создать градиент, используйте разные HEX-коды для каждого слова или буквы.

### 📋 Встроенные переменные (Internal Placeholders)
Эти переменные работают **только внутри конфигурационных файлов** JoinAuction:

**В секции `messages`:**
*   `%name-id-item%` — Отображает имя из `item-settings` или `DisplayName`.
*   `%price-item%` — Цена предмета, указанная при продаже.
*   `%amount%` — Общая сумма прибыли (для оффлайн-сводки).
*   `%count%` — Количество предметов (для оффлайн-сводки).

**В секции `gui` (Lore предметов):**
*   `%item-price%` — Цена лота.
*   `%item-salesman%` — Никнейм продавца.
*   `%item-time%` — Динамический отсчет времени до конца аукциона.
*   `%item-state%` — Статус (Активен/Истек).

## 🛡️ Расширенная логика Blacklist

Система черного списка в `config.yml` работает по трем уровням проверки:

1.  **По материалу (Material):** Блокирует все предметы определенного типа (например, `TNT`).
2.  **По названию (Name):** Блокирует предметы, если их `DisplayName` содержит запрещенное слово.
    *   *Пример:* Если в списке есть "Админ", игрок не сможет продать "Меч Админа".
3.  **По кастомным тегам:** Если вы используете **ItemsAdder** или **Oraxen**, вы можете внести их технические ID в список материалов, и плагин заблокирует их продажу.

## 💾 Техническая информация для администраторов

### Локальное хранение (SQLite)
*   Файл базы данных: `/plugins/JoinAuction/auction.db`.
*   Рекомендуется делать бэкап этого файла раз в сутки.

### Синхронизация (MySQL)
*   Если у вас сеть серверов, используйте одну базу данных MySQL для всех выживаний.
*   **Важно:** Убедитесь, что на всех серверах установлена одинаковая версия плагина и идентичные файлы `lang/ru_ru.yml`, чтобы избежать конфликтов в именах предметов.

### Поддержка кастомных плагинов
*   **MythicMobs:** Сохраняются все уровни, атрибуты и кастомные скиллы на предметах.
*   **EcoEnchants:** Полная поддержка кастомных чар.
*   **ItemsAdder / Oraxen:** Сохранение текстур (CustomModelData) и прочности.
